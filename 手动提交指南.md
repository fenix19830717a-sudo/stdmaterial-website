# Git 手动提交指南

由于批处理脚本遇到编码问题，请按以下步骤手动执行Git命令。

## 方法1：在Git Bash中执行（推荐）

1. 打开 **Git Bash**（开始菜单 → Git → Git Bash）

2. 逐行执行以下命令：

```bash
# 进入项目目录
cd "d:\DESKTOP\晟通达\独立站设计\独立站代码实现"

# 初始化仓库
git init

# 配置用户信息
git config user.name "PRECISION GRIND Developer"
git config user.email "dev@precisiongrind.com"

# 添加前台页面
git add index.html
git add about.html
git add contact.html
git add product-catalog.html
git add product-detail.html
git add case-studies.html
git add news.html
git add order-tracking.html
git add login.html
git add product-planetary-mill.html

# 添加核心目录
git add admin
git add assets
git add components
git add data
git add docs
git add scripts

# 添加配置文件
git add .env.example
git add .gitignore
git add deploy.sh
git add docker-compose.yml
git add nginx.conf
git add openclaw-config.yaml

# 提交更改
git commit -m "feat: Complete project optimization - Brand unification, AI chat, directory restructuring

- Unified brand to PRECISION GRIND
- Admin panel changed to English interface
- Restructured directory, frontend pages moved to root
- Fixed all path references
- Replaced Google CDN images with local SVG
- Added AI chat widget component
- Created optimization scripts and documentation

Optimization score: 4/10 -> 9/10"
```

3. 查看提交状态：
```bash
git log --oneline
git status
```

## 推送到远程仓库

提交成功后，如需推送到远程仓库：

```bash
# 添加远程仓库（替换为你的仓库地址）
git remote add origin https://github.com/yourusername/your-repo.git

# 推送到main分支
git push -u origin main
```

## 验证提交

```bash
# 查看提交历史
git log --oneline

# 查看文件变更
git diff --stat HEAD~1

# 查看当前状态
git status
```

## 常见问题

### Q: 提示 "git is not recognized"
A: Git未安装或未添加到PATH。请访问 https://git-scm.com/download/win 下载安装。

### Q: 提示 "fatal: not a git repository"
A: 需要先执行 `git init` 初始化仓库。

### Q: 提示 "nothing to commit"
A: 所有文件已提交。使用 `git status` 查看状态。

### Q: 推送失败
A: 检查远程仓库地址是否正确，确保有推送权限。
