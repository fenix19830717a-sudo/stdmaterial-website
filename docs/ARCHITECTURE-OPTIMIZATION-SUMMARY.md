## 📋 核心问题

### 当前技术债务

1. **混合技术栈**: React 19 (TypeScript) 与原生 JavaScript 并存 (50+ 文件)
2. **重复代码**: 30+ 个重复文件，代码重复率 40%
3. **配置文件不一致**: Vite 配置、依赖版本冲突
4. **备份文件混乱**: 7 个大型备份目录 (每个 200MB-2GB)
5. **安全隐患**: 45 处 innerHTML 滥用，6 处内存泄漏风险
6. **控制台输出污染**: 100+ console.log 语句

### 架构问题

1. **两套管理后台并存**: 
   - 原有系统：`/admin/` (React JSX)
   - 新增系统：`/admin/login`, `/admin/dashboard` (React TSX)
   
2. **路由冲突**: 自动跳转到新增管理后台

3. **缺乏统一规划**: 组件开发缺乏设计系统指导

## ✅ 方案优势

### 为什么选择渐进式重构而非立即清理？

1. **风险可控**: 渐进式重构可以在每个阶段验证效果，及时发现问题
2. **业务连续**: 不影响现有业务，用户可以正常使用
3. **团队适应**: 给团队充分时间适应新规范和技术栈
4. **质量保障**: 每个阶段都有完整的测试和验收
5. **灵活调整**: 可以根据实际情况调整优先级和时间表

### 为什么选择 Monorepo 架构？

1. **代码复用**: 共享组件和工具函数，避免重复开发
2. **依赖管理**: 统一依赖版本，避免版本冲突
3. **协作效率**: 多个团队可以并行开发不同模块
4. **构建优化**: Turborepo 缓存加速构建
5. **可扩展性**: 方便未来添加新应用和服务
