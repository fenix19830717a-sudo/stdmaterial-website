# 数据库设计文档

## 1. 设计概述

本项目采用前端本地存储方案，使用浏览器的 `localStorage` 进行数据持久化。这种方案适用于 MVP 阶段，无需后端服务器和数据库，简化了部署和维护。

### 1.1 存储方案
- **本地存储**: `localStorage` (浏览器本地存储，容量约5MB)
- **数据格式**: JSON
- **备份方案**: 支持 CSV 导出/导入功能
- **未来扩展**: 可无缝迁移到 MongoDB 等 NoSQL 数据库

## 5. 数据迁移方案

### 5.1 从 localStorage 迁移到 MongoDB

1. **导出数据**: 使用内置的导出功能将数据导出为 JSON 文件
2. **创建 MongoDB 集合**: 对应的数据结构创建集合
3. **导入数据**: 使用 MongoDB Compass 或命令行工具导入数据
4. **修改 API 层**: 更新数据访问层代码，连接到 MongoDB
5. **测试验证**: 确保所有功能正常运行

### 5.2 数据备份策略

1. **自动备份**: 每天自动导出数据到 `backups/` 目录
2. **版本控制**: 备份文件包含时间戳，保留最近7天的备份
3. **手动备份**: 提供手动导出功能，支持 CSV 和 JSON 格式
4. **恢复机制**: 支持从备份文件恢复数据

## 6. 性能优化

### 6.1 前端优化

1. **数据分页**: 大量数据采用分页加载
2. **缓存策略**: 频繁访问的数据使用内存缓存
3. **延迟加载**: 非关键数据延迟加载
4. **批量操作**: 支持批量添加、更新、删除操作

### 6.2 存储优化

1. **数据压缩**: 使用 JSON.minify 压缩存储数据
2. **索引优化**: 维护高效的内存索引
3. **数据清理**: 定期清理过期数据
4. **存储监控**: 监控 localStorage 使用情况，避免超出容量

## 7. 安全性考虑

### 7.1 数据安全

1. **密码加密**: 用户密码使用 bcrypt 加密存储
2. **数据验证**: 所有输入数据进行严格验证
3. **XSS 防护**: 防止跨站脚本攻击
4. **CSRF 防护**: 防止跨站请求伪造

### 7.2 访问控制

1. **角色权限**: 基于角色的访问控制
2. **操作审计**: 记录关键操作日志
3. **会话管理**: 安全的会话管理机制
4. **IP 限制**: 可选的 IP 访问限制

## 9. 结论

本数据库设计文档详细描述了研磨设备 B2B 独立站的数据结构和存储方案。采用 localStorage 作为 MVP 阶段的存储方案，具有以下优势：

1. **快速部署**: 无需后端服务器和数据库
2. **易于维护**: 数据存储在本地文件中
3. **灵活扩展**: 可无缝迁移到传统数据库
4. **成本效益**: 降低初期开发和运维成本

同时，设计中考虑了未来的扩展性和性能优化，为系统的长期发展奠定了基础。
