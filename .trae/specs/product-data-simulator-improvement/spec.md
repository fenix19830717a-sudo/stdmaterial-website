# 产品数据与模拟器功能改进 - 产品需求文档

## Overview
- **Summary**: 改进现有网站的产品数据管理、商品卡交互和研磨模拟器功能，包括使用真实产品图片、修复商品卡跳转、优化模拟器界面和功能。
- **Purpose**: 提升网站的专业性和用户体验，确保产品数据真实准确，增强模拟器的交互性和实用性。
- **Target Users**: 实验室和工业用户，需要研磨设备的潜在客户。

## Goals
- 替换模拟产品数据为真实产品数据，使用无水印的真实图片
- 修复商品卡点击跳转到商品详情页的功能
- 优化模拟器界面，使用拉条控件选择数值
- 增强模拟器功能，提供设备推荐、实时模拟和操作报告

## Non-Goals (Out of Scope)
- 重新设计整个网站的视觉风格
- 实现完整的后端数据库系统
- 开发移动端应用
- 集成第三方支付系统

## Background & Context
- 现有网站使用模拟产品数据，所有产品显示同一张图片
- 商品卡点击功能未实现，无法跳转到详情页
- 模拟器功能基础，但需要更专业的交互和功能
- 目标是提供更真实、专业的用户体验

## Functional Requirements
- **FR-1**: 产品数据更新
  - 从 sbworld.cn 抓取真实产品图片
  - 确保每个产品有唯一的真实图片
  - 保持现有产品分类体系

- **FR-2**: 商品卡交互
  - 实现商品卡点击跳转到对应商品详情页
  - 确保详情页显示正确的产品信息

- **FR-3**: 模拟器界面优化
  - 将数值输入改为拉条（slider）控件
  - 保持三列布局设计
  - 优化UI交互体验

- **FR-4**: 模拟器功能增强
  - 支持用户输入研磨产品类型、细度、产量要求
  - 推荐设备+容器+研磨介质的组合
  - 提供研磨设备视图和模拟研磨动画
  - 支持冷却方案选择
  - 动态展示研磨进度、温度、能耗曲线
  - 提供模拟操作面板（转速控制等）
  - 实现启动/暂停功能
  - 生成操作报告

## Non-Functional Requirements
- **NFR-1**: 性能
  - 模拟器动画流畅运行
  - 页面加载时间不超过3秒
  - 筛选功能响应时间不超过500ms

- **NFR-2**: 用户体验
  - 界面专业、直观
  - 操作流程清晰
  - 响应式设计，适配不同设备

- **NFR-3**: 数据准确性
  - 产品数据真实可靠
  - 模拟器计算基于真实研磨原理

## Constraints
- **Technical**: 前端技术栈保持不变（React + TypeScript + Tailwind CSS）
- **Business**: 保持与 lab-mills.com 产品一致性
- **Dependencies**: 需要从 sbworld.cn 抓取产品图片

## Assumptions
- sbworld.cn 网站可访问且允许抓取产品图片
- 研磨设备的技术参数和计算公式可基于行业标准
- 现有产品分类体系合理，无需修改

## Acceptance Criteria

### AC-1: 产品数据更新
- **Given**: 用户访问产品页面
- **When**: 查看产品列表和详情
- **Then**: 所有产品显示唯一的真实图片，无水印
- **Verification**: `human-judgment`

### AC-2: 商品卡跳转
- **Given**: 用户点击商品卡
- **When**: 点击操作完成
- **Then**: 页面跳转到对应商品的详情页，显示正确信息
- **Verification**: `programmatic`

### AC-3: 模拟器拉条控件
- **Given**: 用户访问模拟器页面
- **When**: 调整参数时
- **Then**: 所有数值输入使用拉条控件，操作直观
- **Verification**: `human-judgment`

### AC-4: 设备推荐功能
- **Given**: 用户输入研磨产品类型、细度、产量
- **When**: 点击计算按钮
- **Then**: 系统推荐合适的设备+容器+研磨介质组合
- **Verification**: `human-judgment`

### AC-5: 研磨模拟动画
- **Given**: 用户启动模拟
- **When**: 模拟运行中
- **Then**: 显示研磨设备视图和动画，实时展示进度、温度、能耗
- **Verification**: `human-judgment`

### AC-6: 操作报告生成
- **Given**: 模拟完成或用户终止
- **When**: 模拟结束
- **Then**: 生成详细的操作报告，包含所有参数和结果
- **Verification**: `human-judgment`
