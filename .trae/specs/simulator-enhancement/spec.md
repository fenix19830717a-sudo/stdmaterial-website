# 模拟器增强 - 产品需求文档

## 概述
- **摘要**: 优化研磨模拟器的可视化效果、设备控制和数据统计功能，使其更接近真实设备操作体验
- **目的**: 解决当前模拟器存在的图表显示、设备控制、可视化和数据统计问题，提供更真实、直观的模拟体验
- **目标用户**: 工程师、技术人员和潜在客户，需要了解设备性能和参数的用户

## 目标
- 修复 TEMPERATURE TREND 和 PARTICLE SIZE DISTRIBUTION 图表的显示问题
- 增加设备模拟控制面板，提供真实设备操作体验
- 优化 Process Visualization 中的介质和材料比例，使其更直观有意义
- 改进 Real-time Data，展示单次模拟过程中的累计数据统计

## 非目标（超出范围）
- 不修改设备推荐逻辑
- 不修改左侧参数配置面板
- 不添加新的设备类型
- 不实现多语言支持

## 背景与上下文
当前模拟器存在以下问题：
1. 温度趋势和粒度分布图表没有正确显示数据
2. 缺少真实的设备控制面板，用户无法直接控制转速、开关等参数
3. 研磨腔可视化中的介质和材料比例不明确，缺乏实际意义
4. 实时数据面板显示静态数据，没有展示模拟过程中的累计变化

## 功能需求
- **FR-1**: 修复 TEMPERATURE TREND 和 PARTICLE SIZE DISTRIBUTION 图表的数据绑定和显示
- **FR-2**: 新增设备模拟控制面板，包含转速调节、开关控制、运行状态等
- **FR-3**: 优化 Process Visualization，明确显示介质和材料的比例关系
- **FR-4**: 改进 Real-time Data，展示单次模拟过程中的累计数据统计（如总能量消耗、平均温度等）

## 非功能需求
- **NFR-1**: 图表更新响应时间小于 100ms
- **NFR-2**: 控制面板操作响应即时
- **NFR-3**: Canvas 动画保持 60fps
- **NFR-4**: UI 在不同屏幕尺寸下保持良好布局

## 约束
- **技术**: 保持使用 React + TypeScript + Chart.js 技术栈
- **业务**: 不破坏现有模拟器的功能
- **依赖**: 继续使用现有的项目依赖库

## 假设
- 用户理解研磨设备的基本操作原理
- Canvas 能够处理更复杂的可视化效果
- Chart.js 能够正确处理动态数据更新

## 验收标准

### AC-1: 图表数据正确显示
- **给定**: 用户启动模拟器
- **当**: 点击 Start 按钮开始模拟
- **则**: TEMPERATURE TREND 和 PARTICLE SIZE DISTRIBUTION 图表应正确显示随时间变化的数据
- **验证**: human-judgment
- **备注**: 图表应有明显的变化趋势

### AC-2: 设备控制面板存在且可用
- **给定**: 用户访问模拟器页面
- **当**: 查看右侧分析中心区域
- **则**: 应存在设备控制面板，包含转速调节、开关控制、运行状态指示等元素
- **验证**: human-judgment

### AC-3: 控制面板操作响应
- **给定**: 用户看到设备控制面板
- **当**: 用户点击开关或调整转速
- **则**: 控制面板应立即响应，Canvas 动画应相应变化
- **验证**: human-judgment

### AC-4: Process Visualization 介质和材料比例明确
- **给定**: 用户查看 Process Visualization
- **当**: 观察研磨腔可视化
- **则**: 介质（研磨球）和材料（粒子）的比例应该清晰可辨，并有意义
- **验证**: human-judgment
- **备注**: 应该能明显看到研磨球和粒子的数量和大小关系

### AC-5: Real-time Data 显示累计统计
- **Given**: 用户启动模拟
- **When**: 模拟运行过程中
- **Then**: Real-time Data 面板应显示单次模拟过程中的累计数据（如总能量消耗、运行时间、平均温度等）
- **Verification**: human-judgment
