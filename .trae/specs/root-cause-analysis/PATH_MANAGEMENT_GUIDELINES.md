# StdMaterial 网站路径管理规范

## 1. 路径类型

### 1.1 绝对路径
- **定义**：以 `/` 开头的路径，从网站根目录开始计算
- **使用场景**：
  - 页面间的跳转链接
  - 重定向路径
  - 外部资源引用（如 CSS、JavaScript 文件）
- **示例**：
  - `/admin/login.html`
  - `/assets/css/main.css`
  - `/api/products`

### 1.2 相对路径
- **定义**：不以 `/` 开头的路径，相对于当前页面的位置
- **使用场景**：
  - 同一目录下的文件引用
  - 子目录文件引用
- **示例**：
  - `dashboard.html`（同一目录）
  - `assets/js/script.js`（子目录）

## 2. 路径使用规则

### 2.1 页面跳转
- **所有页面跳转必须使用绝对路径**
- **禁止**：使用相对路径进行页面跳转，避免路径重复问题
- **示例**：
  - ✅ 正确：`window.location.href = '/admin/dashboard.html';`
  - ❌ 错误：`window.location.href = 'dashboard.html';`

### 2.2 资源引用
- **外部资源（CSS、JavaScript、图片）推荐使用绝对路径**
- **内部资源**：可以使用相对路径，但必须确保路径正确
- **示例**：
  - ✅ 正确：`<link rel="stylesheet" href="/assets/css/main.css">`
  - ✅ 正确：`<script src="./assets/js/script.js"></script>`（同一目录下）

### 2.3 API 调用
- **所有 API 调用必须使用绝对路径**
- **示例**：
  - ✅ 正确：`fetch('/api/products')`
  - ❌ 错误：`fetch('api/products')`

### 2.4 导航链接
- **所有导航链接必须使用绝对路径**
- **示例**：
  - ✅ 正确：`<a href="/admin/products.html">Products</a>`
  - ❌ 错误：`<a href="products.html">Products</a>`

## 3. 目录结构

### 3.1 推荐结构
```
/var/www/html/stdmaterial.com/
├── admin/           # 管理后台
├── assets/          # 静态资源
│   ├── css/         # CSS 文件
│   ├── js/          # JavaScript 文件
│   └── images/      # 图片文件
├── src/             # 源代码
├── index.html       # 首页
└── other pages...   # 其他页面
```

### 3.2 命名规范
- **目录名**：使用小写字母，单词间用连字符 `-` 分隔
- **文件名**：使用小写字母，单词间用连字符 `-` 分隔
- **示例**：
  - ✅ 正确：`product-catalog.html`
  - ❌ 错误：`ProductCatalog.html`

## 4. 服务器配置

### 4.1 Nginx 配置
- **静态文件处理**：使用 `try_files` 指令正确处理静态文件
- **路径重写**：避免不必要的路径重写
- **示例配置**：
  ```nginx
  location /admin {
      try_files $uri $uri/ =404;
  }
  
  location / {
      try_files $uri $uri/ /index.html;
  }
  ```

### 4.2 Express.js 配置
- **静态文件服务**：正确配置静态文件目录
- **路由处理**：避免路径冲突

## 7. 常见问题与解决方案

### 7.1 路径重复问题
- **原因**：使用相对路径进行页面跳转
- **解决方案**：统一使用绝对路径

### 7.2 404 错误
- **原因**：路径拼写错误或文件不存在
- **解决方案**：检查路径拼写和文件存在性

### 7.3 资源加载失败
- **原因**：资源路径错误
- **解决方案**：检查资源路径和文件存在性

## 8. 总结

- **统一使用绝对路径**：避免路径重复和错误
- **保持目录结构清晰**：便于维护和管理
- **定期测试路径**：确保所有路径正常工作
- **遵循命名规范**：保持代码一致性

通过严格遵循本规范，可以有效避免路径相关的问题，提高网站的稳定性和可维护性。
