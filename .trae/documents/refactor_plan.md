# 项目重构计划

## 1. 项目现状分析

### 1.1 已实现的核心系统
- **Logger**：完整的日志系统，支持多级别日志、历史记录和导出功能
- **ErrorHandler**：全局错误处理系统，支持错误报告和用户通知
- **EventBus**：事件总线系统，支持事件监听、触发、移除，以及通配符和一次性监听

### 1.2 待实现的系统
- **StateManager**：状态管理系统，支持状态管理和时间旅行调试
- **ModuleLoader**：模块加载器，支持多模块加载架构
- **商品管理系统**：JSON Schema验证、ProductIndex和FTS搜索
- **研磨模拟器引擎**：物理计算模型
- **现代化UI**：集成shadcn-ui

## 2. 架构设计

### 2.1 核心架构
```
├── assets/
│   ├── js/
│   │   ├── core/           # 核心系统
│   │   │   ├── logger.js
│   │   │   ├── error-handler.js
│   │   │   ├── event-bus.js
│   │   │   ├── state-manager.js
│   │   │   └── module-loader.js
│   │   ├── modules/        # 功能模块
│   │   │   ├── product/
│   │   │   ├── grinder/
│   │   │   └── ui/
│   │   └── app.js          # 应用入口
│   └── css/
│       └── tailwind.css
├── tests/                  # 测试文件
└── index.html              # 主页面
```

### 2.2 模块依赖关系
- **核心系统**：相互独立，提供基础功能
- **功能模块**：依赖核心系统，实现具体业务逻辑
- **应用入口**：整合所有模块，启动应用
