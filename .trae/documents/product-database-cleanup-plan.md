# 产品数据库清理和重构计划

## 目标
根据前面定义的产品详情格式，重新制作数据库中的产品标题和详情内容。删除重复产品，修正非重复产品为英文内容，特别是替换不可用的示例图片。

## 当前问题分析
1. 产品名称混合中英文（如"不锈钢振动粉碎机"）
2. 描述字段混合中英文（如"Professional 不锈钢振动粉碎机 designed for..."）
3. 规格参数字段使用中文（如"处理量："、"进料粒度："）
4. 图片路径使用通用示例图片（如"assets/images/products/gm-ss-8.jpg"）
5. 部分产品可能重复

## 实施步骤

### 步骤 1：分析现有产品数据
- 读取 products.json 文件
- 识别重复产品（基于名称、描述、规格等）
- 统计需要保留的产品数量
- 识别所有使用示例图片的产品

### 步骤 2：定义产品数据标准格式
- 产品名称：统一使用英文名称（nameEn 字段）
- 产品描述：使用完整的英文描述（descriptionEn 字段）
- 规格参数：所有参数字段名使用英文
- 图片：为每个产品生成或指定真实的产品图片
- 分类：统一使用英文分类名称

### 步骤 3：清理重复产品
- 识别重复产品（基于产品名称、规格、描述相似度）
- 保留最完整的产品信息
- 删除重复产品记录
- 记录删除的产品 ID 列表

### 步骤 4：重构产品标题和详情
- 将产品名称统一为英文（使用 nameEn 字段）
- 重写产品描述，确保完整、专业的英文描述
- 将规格参数字段名翻译为英文（如"处理量" → "Capacity"）
- 添加产品详情字段：
  - overview（产品概述）
  - workingPrinciple（工作原理）
  - features（产品特性）
  - technicalParameters（技术参数）
  - accessories（配件信息）

### 步骤 5：替换产品图片
- 识别所有使用通用示例图片的产品
- 为每个产品生成或指定真实的产品图片路径
- 图片命名规范：使用产品 ID 或英文名称
- 确保图片文件实际存在或可访问

### 步骤 6：验证和测试
- 检查所有产品数据的完整性
- 验证图片路径是否正确
- 确保英文内容语法正确、专业
- 测试产品详情页展示效果

## 预期结果
1. 所有产品名称统一为英文
2. 所有描述和详情内容为专业英文
3. 规格参数字段名统一为英文
4. 每个产品都有真实可用的图片
5. 删除重复产品，保持数据库整洁
6. 产品详情包含完整的概述、工作原理、特性、技术参数和配件信息

## 风险控制
- 备份原始 products.json 文件
- 分步骤执行，每步完成后验证
- 保留删除产品的记录，以便需要时恢复
